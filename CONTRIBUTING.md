We welcome contributions from the community! Here are some guidelines to get started:

1. Fork the repository on GitHub.
2. Create a new branch (e.g., `git checkout -b feature-branch`).
3. Make your changes.
4. Test your changes to ensure they work as expected. Please run all existing tests and add new tests if applicable. Itâ€™s important to ensure that all edge cases are covered.

6. Push to your branch (e.g., `git push origin feature-branch`).
7. Open a Pull Request with a detailed description of your changes.

By participating in this project, you agree to abide by our Code of Conduct.
